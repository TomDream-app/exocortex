# Research Program

## 1. Problem

The project aims to build a navigable cognitive landscape that is not reduced to storage, retrieval, or graph logic.
Its ambition is to support an auto-architecturing memory space with intra-spatial relational intelligence.

The current problem is not only technical.
It is to determine which primitives, transformations, and spatial logics can support such a system without flattening it into a database, a chatbot memory, or a conventional knowledge graph.

## 2. Open Questions

- What is the minimal operative primitive of the exocortex?
- What makes a cognitive landscape navigable beyond search and retrieval?
- How should relational intelligence operate inside the space itself?
- How can memory remain structurally active rather than archival?
- What role should negative space, absence, and latent structure play?
- Which transformations are fundamental: extraction, condensation, contrast, composition, displacement, or others?
- What is the first executable layer of the system: capture, extraction, memory, or transformation?

## 3. Research Axes

### Axis 1 — Primitive
Define the elementary unit or pre-unit of the exocortex.

### Axis 2 — Spatiality
Clarify what kind of space is being designed: relational, topological, spectral, negative, morphogenetic.

### Axis 3 — Memory
Determine how memory acts on the architecture instead of merely storing content.

### Axis 4 — Transformation
Identify the core transformations that generate structure from raw material.

### Axis 5 — Navigation
Define what “navigable” means in this context: not only searchability, but perception of proximity, tension, contrast, and latent neighborhoods.

### Axis 6 — Auto-architecture
Explore how the system can progressively reorganize itself through its own operations.

## 4. Transition to Core

The `core/` folder should not be filled before these axes are at least minimally clarified.

- `capture.md` depends on the definition of raw input regimes
- `log_extractor.md` depends on extraction logic and primitive hypotheses
- `memory.md` depends on the status of memory as active structure
- `structure.md` depends on the spatial model of the landscape
- `transformation.md` depends on the list of core operations
- `action.md` depends on what counts as an effective intervention in the space

## 5. Immediate Goal

Use this research program to decide the order of implementation of the `core/` primitives.
The objective is not to solve everything at once, but to determine the first valid operative layer.
