# 07 — Domains

## Role

A domain is a coherent field of material that can be transformed into a navigable cognitive landscape.

A domain is not:
- a core primitive,
- a generic architectural principle,
- or an application by itself.

A domain is a structured source field that can later be:
- explored,
- segmented,
- linked,
- transformed,
- visualized,
- and possibly published in another form.

## Function in Exocortex

Domains provide situated material for the Exocortex architecture.

They allow the project to test whether a cognitive landscape can emerge from:
- experiential notes,
- conceptual fragments,
- recurring forms,
- rhythms,
- paths,
- and structural relations.

## Current status

At this stage, domains are introduced as an intermediate layer between:
- the general architecture (`docs/`),
- the core principles (`core/`),
- and future engines or applications (`apps/`).

## First pilot domain

The first pilot domain is:

- `walking`

It is introduced as a domain because it is already more than a note, but not yet an application or a generic engine.

## Domain criteria

A domain should have:
- a coherent field of inquiry,
- a recognizable internal vocabulary,
- a possible corpus,
- structural recurrence,
- and the potential to be transformed into a navigable system.

## Initial implication

The introduction of domains clarifies the architecture of Exocortex:

- `docs/` = general doctrine and architecture
- `core/` = transversal principles
- `domains/` = situated source fields
- `apps/` = applications or executable branches
